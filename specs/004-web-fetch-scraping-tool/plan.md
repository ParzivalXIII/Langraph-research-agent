# Implementation Plan: Web Fetch and Scraping Tool

## Summary

Add a `WebFetchTool` enrichment layer to the existing retrieval pipeline.  
After Tavily discovers candidate URLs (via `RetrievalService.retrieve_sources()`), 
`WebFetchTool.fetch_batch()` fetches the full page content for each source and 
back-fills `SourceRecord.snippet` with clean markdown (or structured JSON) extracted 
by BeautifulSoup4. Playwright provides an optional JS-rendering path for dynamic pages.

No public API changes, no new routes, no new storage. The enrichment is opt-in via
an `enrich: bool = False` keyword argument added to `retrieve_sources()`.

**Key technology choices** (from [research.md](research.md)):
- **httpx AsyncClient** — native async, already in deps; replaces user-suggested Scrapy
  (Twisted/asyncio conflict makes Scrapy incompatible with uvicorn in-process)
- **BeautifulSoup4** — already in deps; html.parser backend; no C build dep
- **markdownify** — HTML→markdown; only new dep
- **Playwright** — already in deps; async API; opt-in via `use_headless=True`

## Design Decisions

### DD-01: httpx over Scrapy

Scrapy uses a Twisted reactor that conflicts with uvicorn's asyncio event loop when run in-process.
`httpx.AsyncClient` is already a project dependency, natively async, and provides all required
HTTP primitives (redirects, SSL, timeouts, connection pooling). Using Scrapy would require spawning
a subprocess or thread isolation with no benefit for non-crawling single-URL fetches.

*See research.md R-01 for alternatives considered.*

### DD-02: Executor pattern — NOT reused for this tool

Unlike `TavilyTool` (which wraps a sync client via `run_in_executor`), `WebFetchTool` uses
`httpx.AsyncClient` directly. No executor needed. The Playwright path also uses the native
`playwright.async_api` which is natively async. This keeps the fetch pipeline fully in-event-loop.

### DD-03: `retrieve_sources(enrich=False)` — keyword-only opt-in

Adding `enrich` as a keyword-only arg with default `False` maintains 100% backward compatibility
with all existing callers (`ResearchAgent.process_query`). When `True`, the `_enrich_sources()`
private method runs post-scoring. The method imports `WebFetchTool` locally to avoid circular
imports between `services/` and `tools/`.

### DD-04: Per-domain rate limiting via asyncio primitives

A global `asyncio.Semaphore(max_concurrency)` caps parallel fetches. A `dict[str, float]`
(`_last_request_time`) tracks per-domain last-fetch timestamps. Before each request, if
`elapsed < rate_limit_interval`, `await asyncio.sleep(wait)`. State lives entirely within the
`WebFetchTool` instance — which is constructed fresh per `retrieve_sources` call — satisfying
the Stateless Core principle.

### DD-05: Playwright isolation and fallback

Playwright is imported inside `_fetch_with_playwright()` with `try/except ImportError` guard.
If the import or browser launch fails, the method raises `WebFetchError(reason="headless_unavailable")`
and the caller falls back to the plain httpx path. `web_fetch_headless_enabled=False` in
Settings means zero Playwright overhead by default.

### DD-06: SourceRecord mutation is safe

`SourceRecord` fields `snippet` and `retrieved_at` are both `Optional` with no immutability
constraint. Enrichment only writes to a source when fetch succeeds (`page.succeeded`).
Partial failures (some URLs fail) produce graceful degradation — those sources retain their
original Tavily-provided snippet. No exception propagates to the caller.
