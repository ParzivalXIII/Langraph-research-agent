### Ôťů TC-019 / SC-005: URL format validation (FIXED)

**Issue**: Malformed URLs were not validated at schema boundary  
**Fix Applied**: Added regex validation to enforce `^https?://` pattern

```python
# app/schemas/web_fetch.py - Updated validate_urls()
import re  # Added at top

@field_validator("urls")
@classmethod
def validate_urls(cls, v: list[str]) -> list[str]:
    for url in v:
        if not url or not isinstance(url, str):
            raise ValueError("All URLs must be non-empty strings")
        if not re.match(r"^https?://", url, re.IGNORECASE):
            raise ValueError(
                f"URL must be a valid HTTP/HTTPS URL: {url!r} (must start with http:// or https://)"
            )
    return v
```

**Verification**: 8/8 validation test cases pass (rejects malformed, accepts valid URLs)
